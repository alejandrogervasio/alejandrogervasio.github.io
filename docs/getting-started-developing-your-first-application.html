Getting Started: Developing your First Java Application

Published by Alejandro Gervasio On March 24, 2018 Under Java SE

Introduction

Take my word for it: getting started programming in Java doesn't take very long, and it's pretty easy to learn even for beginners, regardless of whether you already have a strong background on object-oriented programming, or just want to learn the paradigm's key concepts with minimal footprint.

To get the ball rolling, let's create a simple example: a naive application that displays in the console the classic "Hello World" message.

One last caveat before we get started: you already must have Java installed on our computer. If you haven't done it yet, please check the official documentation on how to install Java on your operating system.

Defining the "HelloWorld" Class

We're going to develop the application without relying on any IDE in particular, so launch your preferred text editor, e.g. NotePad on Windows or Text Edit on Mac, and write the following code:

public class HelloWorld {
                            
    public static void main(String[] args) {
        System.out.println("Hello World!");
    }
}

Now, save the snippet as "HelloWorld.java" in the location of your choice on the computer.

This is required, of course, as the name of every source file in Java must match the name of its public class and must end with the ".java" extension, so it can be compiled and executed later on, assuming that the file doesn't contain errors.

The Class Implementation, Step by Step

The above example is contrived, but it shows in a nutshell how to create a basic Java program. Nevertheless, let's break it down in several parts, so we can understand the role that plays each one of them:

  1. The HelloWorld class is declared a public class by using the public access modifier and the class keyword, which means that the class can be accessed with no restrictions by any other class.
  2. The main method is, in this case, the only method implemented by the class, and also the entry point of the Java application. The method is declared static by using the static keyword, thus there's no need to instantiate the class to call it. As we did before with the class, the public access modifier is used to declare the method public and the void keyword to specify that it doesn't return a value.
  3. The main method's parameter String[] args: an array of strings, which can be used as command line arguments.
  4. The System.out.println("Hello World!") line just prints out to the console the "Hello World" text and appends a new line at the end.

Compiling the "HelloWorld.java" File

At this stage, "HelloWorld.java" is just a plain text file with a fancy extension. To turn it into an executable Java file, though, first we need to compile it.

Compiling a file means basically converting it to a machine-readable version. In Java, the compiled code is called byte code, and a compiled file has the .class extension.

To compile "HelloWorld.java", open a shell prompt or terminal (of course this varies according to your operating system) and type the "javac" command, followed by the file name (please notice that the .java extension is required):


javac <path/to/HelloWorld.java> HelloWorld.java

Assuming that no compile-time errors were found when compiling the file, a new file called "HelloWorld.class" should have been created in the same location of "HelloWorld.java".

This file contains the compiled version of the class, hence it can be executed.

Running the Executable File

This is where the real fun begins! To run the executable .class file, type the "java" command, followed by the file name (unlike the "javac" command, the .class extension must not be included):


java <path/to/HelloWorld.class> HelloWorld

If all goes well, you should see the "Hello World!" message printed out to the console:


Hello World!

Feel free to pat yourself on the back! You've developed your first Java application.

Refactoring the Example

Of course, there's plenty of room to tweak the example and generate different slants of it. For instance, we could refactor the main method, so it can output to the console a command line argument entered when running the executable file:

public static void main(String[] args) {
    System.out.println(args[0] + " says Hello World!");
}

In this case, compile the source file as usual and execute it passing one single command line argument:


java <path/to/HelloWorld.class> HelloWorld Mary

As expected, the output generated by the file would be the following text:


Mary says Hello World!

While this may seem superfluous, the refactored version of the class uses an important data structure of Java: arrays.

For now, however, don't get ahead of yourself and just keep focused on just practicing all what you've learned so far. It's a lot of ground to cover, trust me.

Conclusion

In this article, we learned how to develop a basic Java application by using just a text editor and a shell prompt. No unnecessary IDE-centric complexities were added to the development process.

There are literally countless examples all around the web that show how to do nearly the same (and even with more options), so that's why I decided to keep this post simple and straight to the point.

See you in the next article. Stay tuned!
